1. The Problem: 

The SparseOcc codebase was written in 2023 for PyTorch 2.0 and CUDA 11.8. You are using an RTX 5070 Ti from 2026.

    The Architecture Gap: Your GPU uses the Blackwell architecture (sm_120). CUDA 11.8 (the original requirement) literally does not know what a 5070 Ti is. 

    The Header Shifting: In 2025/2026, NVIDIA changed how CUDA is packaged. Core C++ files like nv/target were moved into a new sub-package called cuda-cccl. The old SparseOcc build script couldn't find them, leading to that "100 error limit" explosion.

    The Compiler "Nanny" Check: Newer Linux systems come with GCC 14+. CUDA 12.8 (needed for your card) has a built-in safety check that refuses to compile if the GCC version is "too new" (anything over 13), fearing potential code instability.

2. The Fix: The "2026 Blackwell" Stack

We abandoned the 2023 requirements and built a native 2026 environment that speaks your GPU's language.
Step A: Modernizing the Stack

Instead of forcing old PyTorch 2.0, we used PyTorch 2.8+ and CUDA 12.8. These versions natively understand the 5070 Ti and the sm_120 architecture, as confirmed by your torch.cuda.get_arch_list() check.
    # Create and activate a fresh environment
    conda create -n sparseocc python=3.10 -y
    conda activate sparseocc

    # Install the 2026 Blackwell-ready CUDA Toolkit and Compiler
    conda install -c nvidia cuda-toolkit=12.8 cuda-nvcc=12.8 cuda-cccl=12.8 -y

    # Install PyTorch with native CUDA 12.8 support
    pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu128

    # Install a compatible C++ compiler (Avoids the GCC 14 "Too New" error)
    conda install -c conda-forge gxx_linux-64=11 gcc_linux-64=11 -y

Step B: The "CCCL" and Header Injection

We manually installed cuda-cccl and explicitly told the compiler where the "hidden" headers were using CPATH.
    pip install openmim ninja setuptools==59.5.0 numpy==1.23.5
    mim install mmcv-full==1.6.0
    mim install mmdet==2.28.2
    mim install mmsegmentation==0.30.0
    mim install mmdet3d==1.0.0rc6

    # Optional but recommended for data loading speed
    sudo apt-get update && sudo apt-get install -y libturbojpeg
    pip install pyturbojpeg
    pip uninstall -y pillow
    pip install pillow-simd==9.0.0.post1

    This fixed the fatal error: nv/target: No such file or directory that was causing the build to crash at the very start.

Step C: The GCC Downgrade

We installed GCC 11 inside your Conda environment. By setting export CC and export CXX, we tricked the build script into using the older, stable compiler that CUDA 12.8 actually trusts, bypassing the version-check error.
Step D: Targeting sm_120
    cd models/csrc
    rm -rf build/

    # 1. Force use of the compatible GCC 11 compiler
    export CC=$CONDA_PREFIX/bin/x86_64-conda-linux-gnu-gcc
    export CXX=$CONDA_PREFIX/bin/x86_64-conda-linux-gnu-g++

    # 2. Map the 2026-style "sharded" CUDA headers
    export CUDA_HOME=$CONDA_PREFIX
    export PATH=$CONDA_PREFIX/bin:$PATH
    export CPATH="$CONDA_PREFIX/include:$CONDA_PREFIX/targets/x86_64-linux/include:$CPATH"

    # 3. Target the RTX 5070 Ti specifically (Blackwell = sm_120)
    export TORCH_CUDA_ARCH_LIST="12.0"
    export FORCE_CUDA=1

    # 4. Build and Install in editable mode
    pip install --no-build-isolation -e .

By setting export TORCH_CUDA_ARCH_LIST="12.0", we told the compiler: "Don't guess what GPU I have. Build specifically for Blackwell." This ensures that the Sparse Convolution kernels in SparseOcc are optimized for the specific Tensor Cores in your 5070 Ti.
